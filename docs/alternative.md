## 기존 대안

- GitHub Copilot 학습 가이드: inline suggestion을 끄고 Copilot을 튜터처럼 쓰도록 설정하는 방식을 안내한다.
- Cursor Learn: Cursor 대화 기록에서 개념, 패턴, 원칙을 추출해 프로그래밍 레슨으로 바꾸는 로컬 우선 도구다.
- CodeTeach: 코드 스니펫이나 GitHub repo를 입력하면 interactive course, 빈칸 채우기, checkpoint, spaced repetition을 생성한다.
- Exercism: 언어별 연습과 멘토링을 제공한다.
- CodeCrafters / build-your-own-x: 실제 시스템을 직접 재구현하며 학습하게 한다.

## 연구/시장 신호

- METR의 2025년 RCT는 숙련 오픈소스 개발자 16명이 246개 실제 작업을 수행한 결과, AI 허용 조건에서 작업 시간이 19% 증가했다고 보고했다. 개발자들은 사전에는 24% 단축, 사후에는 20% 단축으로 체감했지만 실제 측정은 반대였다.
- 이는 AI 사용이 "느낌상 생산적"이어도 실제 이해, 검토, 유지보수 비용이 따로 존재한다는 신호다.
- vibe coding 확산으로 "코드는 생기지만 이해는 남지 않는" 문제가 더 커지고 있다.

## 비어 있는 틈

일반 코딩 교육 도구는 많다. 하지만 다음 조합은 아직 뚜렷하게 강한 표준 도구가 보이지 않는다.

- 내가 AI와 실제로 만든 코드 변경을 입력으로 삼는다.
- 그 변경에서 내가 이해해야 할 개념과 설계 판단을 추출한다.
- AI가 작성한 코드를 내가 다시 손으로 구현하게 만든다.
- PR/diff/commit 단위로 "내가 진짜 이해했는지"를 테스트한다.
- 학습 결과를 개념 그래프나 누적 맵으로 저장한다.
